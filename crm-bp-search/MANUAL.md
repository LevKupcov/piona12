# Что сделать вручную (после обновления кода)

Кодовая часть доведена до ~95% соответствия ТЗ. Ниже — шаги, которые нужно выполнить на вашей стороне.

## 1. Обязательно сейчас (5 мин)

- [ ] **Переустановить приложение** в Bitrix24 (Разработчикам → Переустановить)
  - Обновятся activity, placement-редактор и новые доп. результаты
- [ ] Проверить `.env`:
  ```
  APP_CLIENT_ID=local.xxx
  APP_CLIENT_SECRET=xxx
  APP_CLIENT_ID и APP_CLIENT_SECRET — для refresh token
  ```
- [ ] **ngrok** должен быть запущен при тестах

## 2. Проверка визуального редактора (10 мин)

- [ ] Открыть дизайнер БП → блок «Поиск CRM по полям»
- [ ] Нажать **шестерёнку / настройки** на блоке — должен открыться **визуальный редактор** (не только JSON)
- [ ] Проверить:
  - выпадающий список **полей** (подгружается из CRM)
  - выпадающий список **условий** (равно, содержит…)
  - кнопка **«+ Добавить условие»**
  - переключатель **AND / OR**
- [ ] Сохранить и запустить БП на тестовой сделке

## 3. Проверка расширенных результатов (10 мин)

В уведомлении или логе БП проверить новые доп. результаты:

| Поле | Что проверить |
|------|----------------|
| `FOUND_IDS` | массив ID |
| `FOUND_COUNT` | число |
| `FOUND_TITLES` | названия сделок |
| `FIRST_FOUND_ID` | ID первой |
| `FOUND_ITEMS` | JSON с полями (если задан SELECT_FIELDS) |

Пример уведомления:
```
Найдено: {=...:FOUND_COUNT}
Первый ID: {=...:FIRST_FOUND_ID}
```

## 4. Тестовые данные в CRM

Создайте сделки:

| Название | Попадёт при поиске «тест» |
|----------|---------------------------|
| Тестовая сделка | да |
| Договор тест №1 | да |
| Просто тест | да |
| Договор без совпадения | нет |

Условие в редакторе: поле `TITLE`, условие `содержит`, значение `тест` → ожидается **3**.

## 5. Публикация в Маркет (когда будете готовы)

Это **нельзя** сделать из кода — только вручную:

- [ ] Аккаунт [dev.1c-bitrix.ru](https://dev.1c-bitrix.ru) — **публичное** приложение (не локальное)
- [ ] Стабильный **HTTPS-хостинг** (VPS, не ngrok)
- [ ] Описание, иконка 256×256, скриншоты дизайнера БП
- [ ] Политика обработки данных
- [ ] Тестовый портал для модерации
- [ ] Прайс (бесплатно / платно)

## 6. Опционально (улучшения UX)

Не блокирует сдачу, но можно доделать позже:

- [ ] Кнопка «Вставить значение» как в стандартном БП (нативный диалог Bitrix в placement)
- [ ] Автовыбор поля `TITLE`/`title` при смене сущности
- [ ] Локализация EN для placement-формы

## 7. Если что-то не работает

| Проблема | Решение |
|----------|---------|
| Нет визуального редактора | Переустановить приложение |
| «Ошибка загрузки полей» | Права `crm`, ngrok активен |
| БП зависает | handler.php недоступен — проверить ngrok |
| `expired_token` | Заполнить CLIENT_ID/SECRET в `.env` |

**После переустановки** откройте блок в дизайнере — настройки должны открываться в слайдере с формой условий.
